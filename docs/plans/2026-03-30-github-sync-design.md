# Obsidian Git Sync 插件设计文档

## 项目概述

### 插件名称
Obsidian Git Sync

### 核心目标
为 Obsidian 用户提供免费、便捷的多设备知识库同步方案，通过 GitHub 私有仓库实现跨平台同步（电脑、手机、平板）。

### 目标用户
- 已有 GitHub 账号，日常使用 Git 的用户
- 需要在多设备间免费同步知识库
- 设备类型：电脑、手机、平板（多平台混合）

### 核心需求
- 免费：通过 GitHub 私有仓库实现
- 多设备：支持电脑、iOS、Android
- 自动同步：减少手动操作
- 单文件级别同步：首次全量，后续增量

## 认证与仓库管理

### GitHub 认证流程

使用 Personal Access Token 认证：
1. 插件引导用户前往 GitHub 创建 Personal Access Token
2. 需要权限：`repo`（私有仓库访问）、`user`（用户信息）
3. 用户粘贴 Token，插件验证后加密保存

### 仓库管理流程

#### 首次使用

```
用户选择：
├── 创建新仓库
│   ├── 输入仓库名（默认：obsidian-vault）
│   ├── 选择可见性（强制私有）
│   └── 插件调用 GitHub API 创建
│
└── 使用已有仓库
    ├── 列出用户私有仓库供选择
    └── 或输入仓库 URL
```

#### 首次全量同步
1. 确认仓库为空或允许覆盖
2. 扫描 Vault 所有文件
3. 应用排除规则过滤文件
4. 检查文件大小（默认阈值 100MB）
5. 创建初始 commit 并推送

## 同步机制

### 双向同步流程（bidirectionalSync）

```
┌─────────────────────────────────────────────────────────┐
│                   bidirectionalSync()                    │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  第一步：pullFromRemote() 拉取远程变更                    │
│  ┌─────────────────────────────────────────────────────┐ │
│  │ 1. 获取远程所有文件列表                              │ │
│  │ 2. 遍历远程文件：                                    │ │
│  │    - 远程有、本地无 → 下载创建                        │ │
│  │    - 远程有、本地有 → 检查冲突后更新                  │ │
│  │ 3. 遍历本地文件：                                    │ │
│  │    - 本地有、远程无 → 删除本地文件                    │ │
│  └─────────────────────────────────────────────────────┘ │
│                         ↓                                │
│  第二步：检查冲突                                        │
│  ┌─────────────────────────────────────────────────────┐ │
│  │ if (conflicts.length > 0) {                         │ │
│  │     暂停同步，通知用户                               │ │
│  │     return                                          │ │
│  │ }                                                   │ │
│  └─────────────────────────────────────────────────────┘ │
│                         ↓                                │
│  第三步：推送本地变更                                    │
│  ┌─────────────────────────────────────────────────────┐ │
│  │ 扫描本地所有文件，对比远程 SHA：                      │ │
│  │ - 远程不存在 → 上传                                  │ │
│  │ - 本地有修改 → 上传                                  │ │
│  │ - 已同步 → 跳过                                      │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 同步触发时机

#### 自动同步
- 文件创建/修改时：自动上传到 GitHub
- 文件删除时：自动从 GitHub 删除
- 文件移动/重命名时：删除旧路径 + 上传新路径
- 启动时同步：Obsidian 启动时自动拉取远程变更（可配置）

#### 手动同步
- 状态栏点击菜单
- 命令面板命令（可设置快捷键）

### 本地操作同步映射

| 本地操作 | 事件监听 | 同步动作 |
|----------|----------|----------|
| 新建文件 | `vault.on('create')` | 上传到 GitHub |
| 修改文件 | `vault.on('modify')` | 更新 GitHub 文件 |
| 删除文件 | `vault.on('delete')` | 从 GitHub 删除 |
| 移动/重命名 | `vault.on('rename')` | 删除旧路径 + 上传新路径 |

### 远程操作同步映射

| 远程操作 | 检测方式 | 同步动作 |
|----------|----------|----------|
| 新建文件 | 远程有、本地无 | 下载到本地 |
| 修改文件 | SHA 不同 | 更新本地文件 |
| 删除文件 | 本地有、远程无 | 删除本地文件 |

### 临时文件过滤

以下文件名会被自动过滤，不同步到 GitHub：
- `未命名`、`未命名-1`、`未命名-2`...
- `Untitled`、`Untitled-1`、`Untitled-2`...
- `New note`、`新笔记`

### 单文件同步流程

```
文件修改触发同步：
├── 检查文件是否在排除列表 → 跳过
├── 检查文件大小（100MB） → 超限则提示
├── 拉取远程该文件最新版本
│   ├── 远程无变更 → 直接推送
│   └── 远程有变更 → 触发冲突处理
└── 推送本地变更
```

### 排除规则配置

#### 预设排除规则
```
.obsidian/plugins/obsidian-git-sync/  # 插件自身
.obsidian/workspace.json              # 工作区状态
.obsidian/workspace-mobile.json
.trash/                               # 回收站
```

#### 可视化配置界面
- 文件类型开关：`.gitignore`、`.obsidian`、`.trash` 等
- 目录树选择器：勾选/取消同步的目录
- 文件扩展名过滤器：如不同步 `.mp4`、`.zip` 等

### 文件大小限制
- 默认阈值：100MB（与 GitHub API 限制一致）
- 用户可在设置中调整
- 超过阈值时跳过并提示用户

## 冲突处理机制

### 智能合并策略

当同一文件在多个设备被修改时：

```
检测到冲突时：
├── 分析变更类型
│   ├── 不同区域修改 → 自动合并
│   ├── 相同区域修改 → 创建冲突副本
│   └── 一方删除一方修改 → 提示用户选择
│
├── 自动合并规则（Markdown 文件）
│   ├── 按段落为单位检测冲突
│   ├── 新增内容直接合并
│   ├── 删除内容需确认
│   └── 修改内容对比差异
│
└── 冲突副本命名
    └── filename_conflict_20260330_143022.md
```

### 冲突通知与解决

#### 状态栏提示
- 显示冲突文件数量
- 点击跳转到冲突文件列表

#### 冲突解决面板
```
┌──────────────────────────────────────┐
│  检测到 2 个文件存在冲突              │
├──────────────────────────────────────┤
│  📄 笔记A.md                         │
│     ○ 保留本地版本                   │
│     ○ 使用远程版本                   │
│     ● 保留两者（重命名远程版本）      │
│     [查看差异]                       │
├──────────────────────────────────────┤
│  📄 笔记B.md                         │
│     ● 已自动合并                     │
│     [查看合并结果]                   │
├──────────────────────────────────────┤
│           [应用所选操作]              │
└──────────────────────────────────────┘
```

### 二进制文件冲突
- 图片、PDF 等无法自动合并
- 默认保留较新的版本（基于修改时间）
- 提示用户选择保留哪个版本
- 可选择保留两者（重命名其中一个）

## GitHub API 使用方案

**不依赖本地 Git 命令**（移动端 Obsidian 无法执行 Git 命令），直接使用 GitHub REST API：

| 操作 | API | 说明 |
|------|-----|------|
| 创建仓库 | `POST /user/repos` | 首次初始化 |
| 获取文件 | `GET /repos/{owner}/{repo}/contents/{path}` | 拉取文件 |
| 上传文件 | `PUT /repos/{owner}/{repo}/contents/{path}` | 推送文件 |
| 删除文件 | `DELETE /repos/{owner}/{repo}/contents/{path}` | 删除同步 |
| 获取目录 | `GET /repos/{owner}/{repo}/contents/{path}` | 列出文件 |
| 比较 commit | `GET /repos/{owner}/{repo}/compare/{base}...{head}` | 检测变更 |

### 同步状态追踪

本地存储同步状态，避免重复同步：

```json
{
  "lastSyncTime": "2026-03-30T14:30:00Z",
  "lastCommitSha": "abc123...",
  "fileStates": {
    "笔记/每日笔记.md": {
      "localSha": "def456...",
      "remoteSha": "def456...",
      "localModified": "2026-03-30T14:25:00Z",
      "status": "synced"
    }
  }
}
```

- **电量优化**：避免频繁轮询，减少电量消耗

## 技术风险点

1. **移动端 OAuth 流程**：需要测试 Obsidian 移动端环境的网络限制
2. **大文件上传**：GitHub API 单文件限制 100MB，已设置阈值控制
3. **冲突合并算法**：Markdown 段落级别合并需要充分测试
4. **API 限流**：GitHub API 有速率限制，需要合理设计同步频率
